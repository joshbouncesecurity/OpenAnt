Generate a security disclosure document from the provided vulnerability data.

INPUT DATA:
{vulnerability_data}

OUTPUT FORMAT:

# Security Disclosure: {short_title}

**Product:** {product_name}
**Type:** CWE-{cwe_id} ({cwe_name})
**Affected:** {affected_versions}

## Summary

{2-3 sentence description of the vulnerability. What is the bug? What can an attacker do?}

## Vulnerable Code

`{file_path}`:

```{language}
{minimal_code_snippet_with_vulnerability_highlighted}
```

{1 sentence explaining why this code is vulnerable}

## Steps to Reproduce

**Prerequisites:**
{bullet list of what attacker needs}

**Step 1:** {action}

{code block if needed}

**Step 2:** {action}

{code block if needed}

[Continue for all steps]

**Tested:** {platform/version}, {date}.

## Impact

- {impact point 1}
- {impact point 2}
- {impact point 3}

## Suggested Fix

{1 sentence description}

```{language}
{fixed_code_snippet}
```

---

Discovered via static analysis.{" Confirmed via dynamic testing." if dynamic_testing else ""}

---

INSTRUCTIONS:
- Keep summary under 50 words
- Code snippets: show only the vulnerable function/section, not entire files
- Steps to Reproduce: if dynamic_testing data exists, use those steps; otherwise write "[REQUIRES DYNAMIC TESTING]" for each step
- Impact: 3-5 bullet points, no sub-bullets
- Suggested Fix: show minimal code change, use comments to indicate what was added
- Do not include CVSS scores or severity ratings
- Do not include disclosure timelines
- Do not include references section
